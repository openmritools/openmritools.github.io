About Open MRI Tools

Suggest a tool GitHub

Great tools shouldn't stay hidden.

The open science MRI community builds incredible tools — but they're scattered across GitHub orgs, lab websites, and word of mouth. Great work gets missed. This directory exists so it doesn't have to.

Open MRI Tools is a shared overview of open-source software for MRI analysis. If everyone contributes what they know, it becomes a way for the whole community to stay updated and discover each other's work.

How health status works. A weekly GitHub Actions cron job checks each tool's repository.

  • Active — committed within 6 months
  • Quiet — 6 to 24 months
  • Inactive — 2+ years or archived
  • Not on GitHub

What gets included. Tools need to be MRI-relevant, open-source, and actively maintained. New or low-star tools are welcome as long as they solve a real problem and are being worked on.

How to contribute. Use the form at /contribute — your submission becomes a public GitHub Issue automatically. You can also open an issue directly on GitHub, or submit a pull request. Each tool is a YAML entry: name, url, description, language, modality, optionally github (owner/repo for health tracking), and added_by to credit whoever suggested it.

Started by Henrik Formoe (University of Oslo). Open to everyone.
